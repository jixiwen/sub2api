# Brainstorm Summary

- Change: image-studio-async-queue
- Date: 2026-06-11

## 确认的技术方案

新增一套仅供后台 `image-studio` 使用的异步生图任务接口，完全保留现有对外同步生图 API 的行为不变。后端采用数据库持久化任务表和应用内 worker，实现“每用户 FIFO + 全局最大并发”的调度；单任务单张图，支持文生图和图生图。成功任务将原图和缩略图保存到单实例本地文件系统，列表展示缩略图，详情按需拉取原图。前端缓存原图，后续预览和下载复用缓存。

## 关键取舍与风险

- 取舍：首期使用单表任务模型和本地文件存储，避免引入对象存储和多结果模型复杂度。
- 取舍：只新增后台专用接口，不改公网同步接口，确保外部 API 零回归。
- 风险：本地文件和应用内 worker 假设单实例部署；后续多实例需迁移共享存储或对象存储。
- 风险：预扣费、失败退款和崩溃恢复需要严格幂等，避免重复扣款或重复退款。
- 风险：前端缓存的原图可能在后端过期清理后变陈旧，需要以后端资源状态为准主动清理缓存。

## 测试策略

- 后端单测覆盖任务创建、状态流转、预扣费/核销/退款、心跳恢复、调度公平性、缩略图生成和过期清理。
- API 测试覆盖后台专用提交、列表、详情、资源访问和余额不足等失败路径。
- 前端测试覆盖异步任务提交、轮询列表/详情、缩略图优先展示、过期资源状态和原图缓存下载复用。
