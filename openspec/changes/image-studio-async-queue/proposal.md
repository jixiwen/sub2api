## Why

站内登录后的 `image-studio` 当前直接调用同步生图接口，容易在上游限流或瞬时高并发时失败，也无法由服务端统一控制总体并发。需要新增一套仅供站内登录用户使用、且不对普通 API 客户端开放的异步任务队列，提升成功率并保留现有对外 API 的兼容性。

## What Changes

- 新增站内登录用户专用的异步生图任务接口，用于提交、查看列表和查看详情，不修改现有对外同步生图接口。
- 新增持久化任务模型、后台 worker 和“每用户 FIFO + 全局并发上限”调度机制。
- 新增站内生图预扣费、成功核销、失败退款的任务结算流程。
- 新增本地文件落盘的原图与缩略图存储、详情按需加载原图、前端缓存原图与下载复用缓存的交互模式。
- 新增全站统一的图片保留时长设置，支持小时/天两个单位，`0` 表示永不过期，并由后台定时清理过期图片文件。

## Capabilities

### New Capabilities
- `image-studio-async-jobs`: 站内登录用户异步生图任务的提交、排队、执行、计费、存储、查询与过期清理能力。

### Modified Capabilities
- 无

## Impact

- 影响后台 Go 服务中的站内用户接口、任务调度、计费、文件存储与定时清理逻辑。
- 影响前端 `frontend/src/extensions/image-studio/*`，从同步请求改为异步任务提交与轮询。
- 不影响现有 `/v1/images/generations`、`/v1/images/edits` 及其他对外 API 契约。
